---
description: Inicializa el contexto del proyecto analizando documentación, estructura y configuración clave. Ideal para comenzar una nueva sesión.
---

Para contextualizarte con el proyecto, por favor realiza los siguientes pasos:

1. **Lee la documentación principal** para entender las reglas de negocio y arquitectónicas:
   - Lee `CLAUDE.md` (Reglas globales y principios).
   - Lee `README.md` (Visión general).
   - Lee `package.json` (Dependencias y scripts).

2. **Analiza la estructura de directorios** para entender la organización "Feature-First":
   - Lista el contenido de `src/app`
   - Lista el contenido de `src/features`
   - Lista el contenido de `src/shared`

3. **Identifica y lee componentes clave** basándote en lo encontrado (ej. `src/app/layout.tsx`, configuración de bases de datos o stores principales).

4. **Genera un Resumen Ejecutivo** con el siguiente formato:
   - **Propósito**: Breve descripción del proyecto.
   - **Arquitectura**: Tipo (Next.js App Router), Patrón (Feature-First), etc.
   - **Stack Tecnológico**: Frontend, Backend, Styling, State, Testing.
   - **Mapa de Features**: Lista rápida de las features encontradas en `src/features`.
   - **Estado Actual**: Resumen de scripts disponibles y dependencias clave.

5. **Confirma que estás listo** preguntando en qué tarea desea trabajar el usuario.
